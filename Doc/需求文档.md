# NewLife.MySql 需求

## 1. 背景与目标

### 背景

- 现有主流 MySQL 驱动存在以下痛点：
  - **MySql.Data (Oracle)**：GPL 许可协议，商用需付费；异步方法为 sync-over-async 伪异步；代码量庞大
  - **MySqlConnector**：功能丰富但非国产自主可控，不满足信创/国产化替代需求
  - 两者均缺少管道化批量执行、数组绑定等大数据批量操作能力
- NewLife 团队已有成熟的 XCode ORM 生态，需要一个原生适配的 MySQL 驱动
- 国产化替代、信创环境对组件的自主可控和许可协议有严格要求

### 目标

- 实现一个**纯国产、零第三方依赖**的高性能 MySQL ADO.NET 驱动
- 覆盖 ADO.NET 标准的增删改查、事务、参数化查询、存储过程和批量操作
- 大数据批量操作场景性能领先竞品 **2×~3×**
- 支持 `net45` ~ `net10` 全版本，MySQL 5.x ~ 9.0+ 全系列
- MIT 开源协议，商用无忧

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|---------|
| XCode 开发者 | 使用 NewLife.XCode ORM 的开发者 | 无缝集成 XCode，开箱即用，无需关心底层驱动 |
| ADO.NET 开发者 | 直接使用 ADO.NET 接口的开发者 | 标准接口、参数化查询、事务、存储过程等完整支持 |
| 大数据开发者 | 需要批量导入/更新/删除大量数据的开发者 | 高性能批量操作，万级到百万级行的快速处理 |
| 信创项目负责人 | 国产化替代项目的技术负责人 | 纯国产、MIT 协议、零第三方依赖、安全审计通过 |
| EF Core 开发者 | 使用 Entity Framework Core 的开发者 | 通过 EF Core Provider 使用 NewLife.MySql 驱动 |

## 3. 功能需求

### 3.1 连接管理

- **描述**：建立和管理 MySQL TCP 连接，支持连接池自动复用
- **用户故事**：作为 ADO.NET 开发者，我希望通过连接字符串快速建立 MySQL 连接，以便执行数据库操作
- **验收条件**（AC）：
  - [x] 支持 MySQL 5.x / 8.0 / 9.0+ 全版本连接
  - [x] 支持 `mysql_native_password` 认证
  - [x] 支持 `caching_sha2_password` 认证（含 RSA 公钥全量认证）
  - [x] 支持 Auth Switch 认证方式切换
  - [x] 支持连接字符串参数解析（Server/Port/Database/UserID/Password 等）
  - [x] 支持连接超时（ConnectionTimeout）和命令超时（CommandTimeout）
  - [x] 支持 SSL/TLS 加密连接（SslMode=None/Preferred/Required）
  - [x] 支持 TLS 1.2 和 TLS 1.3（.NET 5.0+）
  - [x] 支持 `ChangeDatabase` 切换数据库
  - [x] 支持 `COM_INIT_DB` 低级 API 切换数据库
- **优先级**：Must

### 3.2 连接池

- **描述**：按连接字符串自动分池管理连接，支持健康检查和空闲回收
- **用户故事**：作为开发者，我希望连接池自动管理连接复用，以便减少连接建立开销
- **验收条件**（AC）：
  - [x] 默认开启连接池，无需额外配置
  - [x] 按连接字符串哈希分池，相同字符串共用连接池
  - [x] 获取连接时自动验证可用性（Reset + Active 状态检查）
  - [x] 空闲超时 30 秒自动回收，全空闲超时 300 秒
  - [x] 服务器变量缓存 10 分钟，减少 `SHOW VARIABLES` 查询
  - [x] 归还连接时保持打开以便复用
- **优先级**：Must

### 3.3 查询执行

- **描述**：支持 ADO.NET 标准的 SQL 查询执行接口
- **用户故事**：作为 ADO.NET 开发者，我希望使用标准接口执行 SQL 查询，以便读取和操作数据
- **验收条件**（AC）：
  - [x] `ExecuteReader` / `ExecuteReaderAsync` 返回结果集
  - [x] `ExecuteScalar` / `ExecuteScalarAsync` 返回单值
  - [x] `ExecuteNonQuery` / `ExecuteNonQueryAsync` 返回影响行数
  - [x] 支持多结果集遍历（`NextResult`）
  - [x] 支持全链路 `async/await` 真异步
  - [x] DataReader 正确映射 MySQL 数据类型到 .NET 类型
- **优先级**：Must

### 3.4 参数化查询

- **描述**：客户端参数替换，`@参数名` 语法，自动转义防注入
- **用户故事**：作为开发者，我希望使用参数化查询防止 SQL 注入，以便保障数据安全
- **验收条件**（AC）：
  - [x] 支持 `@参数名` 语法绑定参数
  - [x] 自动转义特殊字符（单引号、反斜杠、换行、NULL 字节等）
  - [x] 跳过字符串字面量中的 `@`/`?`（避免误替换）
  - [x] 支持 String / Int32 / Int64 / Boolean / DateTime / Byte[] / Guid / Enum / null 等类型
  - [x] 支持 `DBNull.Value` 传递 NULL
- **优先级**：Must

### 3.5 预编译语句

- **描述**：通过 `Prepare()` 走 `COM_STMT_PREPARE` 二进制协议，支持全局开关
- **用户故事**：作为开发者，我希望预编译重复执行的 SQL 语句，以便减少服务端解析开销
- **验收条件**（AC）：
  - [x] `Prepare()` 发送 `COM_STMT_PREPARE`，获取 `statementId`
  - [x] 后续执行走 `COM_STMT_EXECUTE` 二进制协议
  - [x] 支持修改参数后重复执行（无需重新编译）
  - [x] 支持 `UseServerPrepare=true` 连接字符串全局开关
  - [x] 批量操作自动 Prepare / Unprepare
- **优先级**：Must

### 3.6 事务支持

- **描述**：支持 ADO.NET 标准事务接口，四种隔离级别
- **用户故事**：作为开发者，我希望使用事务保障数据一致性，以便多条 SQL 操作原子执行
- **验收条件**（AC）：
  - [x] `BeginTransaction` / `Commit` / `Rollback` 标准接口
  - [x] 支持 ReadUncommitted / ReadCommitted / RepeatableRead / Serializable 四种隔离级别
  - [x] Dispose 时未提交也未回滚则自动回滚
- **优先级**：Must

### 3.7 存储过程

- **描述**：通过 `CommandType.StoredProcedure` 调用存储过程
- **用户故事**：作为开发者，我希望调用 MySQL 存储过程并获取输出参数，以便使用服务端业务逻辑
- **验收条件**（AC）：
  - [x] 支持 `CommandType.StoredProcedure` 设置
  - [x] 支持 IN 输入参数
  - [x] 支持 OUT 输出参数（调用后读取）
  - [x] 内部转换为 `SET @p=val; CALL proc(@p1,@p2); SELECT @p2` 多语句执行
- **优先级**：Must

### 3.8 批量操作 — 字典参数集

- **描述**：同一 SQL 绑定多组字典参数执行
- **用户故事**：作为大数据开发者，我希望用字典参数集批量执行 DML，以便灵活传入动态参数
- **验收条件**（AC）：
  - [x] `ExecuteBatch` / `ExecuteBatchAsync` 接受 `IList<IDictionary<String, Object?>>` 参数

  - [x] 返回累计影响行数
- **优先级**：Must

### 3.9 批量操作 — 数组绑定

- **描述**：Oracle 风格的数组绑定批量执行
- **用户故事**：作为大数据开发者，我希望用数组绑定批量执行 DML，以便高效处理大批量数据
- **验收条件**（AC）：
  - [x] `ExecuteArrayBatch` / `ExecuteArrayBatchAsync` 接受执行次数参数

  - [x] 返回累计影响行数

### 3.10 批量操作 — 管道化执行

- **描述**：真管道化批量执行，批量发送后批量接收，网络延迟仅一次
- **用户故事**：作为大数据开发者，我希望使用管道化执行批量 DML，以便在高延迟网络下获得数倍性能提升
- **验收条件**（AC）：

  - [x] Phase 1 批量构建并发送所有 EXECUTE 包，仅最后一个包 Flush
  - [x] Phase 2 按顺序批量读取所有 OK/Error 响应，累加影响行数
  - [x] 错误处理：某个命令失败时继续读取后续响应保持连接干净，最后抛出第一个错误
  - [x] 搭配字典参数集和数组绑定均可使用

## 4. 非功能需求

### 性能

- 全链路真异步 IO，无 sync-over-async 反模式
- 使用 `ArrayPool<T>` 和 `OwnerPacket` 管理内存，零额外分配，减少 GC 压力
- 使用 `Pool.StringBuilder` / `Pool.MemoryStream` 全面池化
- 管道化批量执行本机 1,000 行 INSERT 加速比 ≥ 4×，10,000 行批量操作领先竞品 2×~3×
- 精简协议解析，只解析必要字段，跳过冗余数据

### 安全

- 参数化查询自动转义，防止 SQL 注入
- 支持 SSL/TLS 加密连接，支持 TLS 1.2 / 1.3
- 支持 `caching_sha2_password` 认证（SHA256 + RSA 全量认证）
- MIT 协议，无 GPL 许可风险
- 不暴露内部实现路径和敏感信息

### 兼容性

- 目标框架：`net45` / `net461` / `netstandard2.0` / `netstandard2.1` / `net6.0` / `net10.0`
- MySQL 版本：5.5 ~ 9.0+ 全版本
- 兼容数据库：MySQL、OceanBase、TiDB、MariaDB（基础）
- 仅依赖 `NewLife.Core`（同为 NewLife 团队出品），零第三方依赖
- 条件编译针对不同框架自动选择最优实现

## 5. 边界与约束

### 不做什么（明确排除项）

- ❌ 压缩协议（内网部署为主，压缩收益有限）
- ❌ 负载均衡 / 故障转移（应由中间件或 XCode 层处理）
- ❌ LOAD DATA LOCAL INFILE（安全风险，场景少）
- ❌ DbDataSource (.NET 7+)（目标用户使用 XCode/DAL）
- ❌ `sha256_password` 认证（已被 MySQL 8.0 废弃）
- ❌ `ed25519` 认证（MariaDB 专属，非目标场景）
- ❌ 多因子认证 MFA 完整流程（仅协商标志）

### 已知限制

- `ChangeDatabase` 通过 Close/Reopen 实现，不支持在事务中途切换数据库
- 客户端参数替换模式下，SQL 长度受 `max_allowed_packet` 限制
- 管道化执行仅适用于 DML 批量操作，不适用于 SELECT 查询

### 经验总结

- **做得好的**：管道化执行是独创能力，大数据批量场景性能领先；零第三方依赖架构干净；多目标框架覆盖 net45~net10
- **待改进的**：可考虑后续增加连接字符串参数控制连接池大小；可考虑支持更多 Schema 信息查询

## 8. 术语表

| 术语 | 定义 |
|------|------|
| Protocol Version 10 | MySQL Client/Server 二进制通信协议版本号，从 MySQL 3.21 沿用至今 |
| COM_QUERY | MySQL 文本协议查询命令（命令码 0x03），发送 SQL 文本 |
| COM_STMT_PREPARE | MySQL 预编译命令（命令码 0x16），服务端解析 SQL 并返回 statementId |
| COM_STMT_EXECUTE | MySQL 预编译执行命令（命令码 0x17），发送 statementId + 二进制参数 |
| COM_INIT_DB | MySQL 切换数据库命令（命令码 0x02），等效于 USE 语句 |
| mysql_native_password | MySQL 4.1 引入的认证方式，SHA1 双重哈希，20 字节响应 |
| caching_sha2_password | MySQL 8.0 默认认证方式，SHA256 哈希，首次需 RSA/TLS 传输密码 |
| Auth Switch | 服务器返回 0xFE 包要求客户端切换认证方式 |
| Pipeline | 管道化执行模式，批量发送请求后批量接收响应，减少网络往返延迟 |
| ArrayBind | Oracle 风格的数组绑定，参数 Value 设为数组，指定执行次数 |
| DbBatch | .NET 6+ ADO.NET 标准的批量命令 API |
| XCode | NewLife 团队出品的轻量级 ORM 框架 |
| DAL | Data Access Layer，XCode 的数据访问层 |
| OwnerPacket | NewLife.Core 提供的内存包管理器，基于 ArrayPool 实现零分配 |
| MoSCoW | 优先级分类法：Must / Should / Could / Won't |
| OceanBase | 蚂蚁集团出品的分布式关系数据库，兼容 MySQL 协议 |
| TiDB | PingCAP 出品的分布式关系数据库，兼容 MySQL 协议 |
